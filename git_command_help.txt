1. Початок роботи
Ці команди використовуються для створення нового репозиторію або копіювання існуючого.

git init — ініціалізує новий локальний репозиторій у поточній папці.

git clone <url> — копіює віддалений репозиторій на ваш комп'ютер.

2. Робота зі змінами (Staging & Committing)
Процес збереження вашої роботи складається з двох етапів: підготовки файлів та самого запису змін.

git status — показує стан робочої директорії (які файли змінені, а які готові до коміту).

git add <file> — додає конкретний файл до області підготовки (staging area).

git add . — додає всі змінені та нові файли до підготовки.

git commit -m "опис змін" — створює знімок (коміт) підготовлених змін із коротким повідомленням.

git diff — показує різницю між робочим каталогом і останнім комітом (що саме ви змінили в коді).

3. Гілки (Branching)
Гілки дозволяють працювати над новими функціями окремо від основного коду.

git branch — список усіх локальних гілок.

git branch <name> — створення нової гілки.

git checkout <name> — перехід на вказану гілку.

git checkout -b <name> — створення нової гілки та миттєвий перехід на неї.

git merge <name> — злиття вказаної гілки з поточною (зазвичай виконується на гілці main).

4. Синхронізація з сервером (Remote)
Використовується для обміну кодом із GitHub, GitLab або іншими сервісами.

git remote add origin <url> — прив’язка локального репозиторію до віддаленого.

git push -u origin <branch> — відправка локальних комітів на сервер (прапорець -u запам'ятовує гілку для майбутніх пушів).

git pull — завантаження змін із сервера та їх автоматичне злиття з вашим кодом.

git fetch — завантаження даних із сервера без злиття (корисно, щоб просто подивитися, що змінили колеги).

5. Скасування та виправлення
Коли щось пішло не так.

git checkout -- <file> — скасовує зміни у файлі до стану останнього коміту.

git reset --hard HEAD — повне скасування всіх незамічених змін (обережно, дані видаляються назавжди!).

git commit --amend -m "новий опис" — дозволяє змінити повідомлення останнього коміту або додати в нього забуті файли.

git log --oneline — перегляд історії комітів у компактному вигляді.
